NAnt Help Task Reference <add-access-rule> @NAntConsoleVersion@

<add-access-rule>

Adds access rules (authorization) to files and directories.

Parameters

Attribute Type Description Required
dir Directory Directory to apply changes to. False
file File File to apply changes to. False
share String The network share to apply changes to. (only FullControl is working). False
ntaccount String NTAccount (DOMAIN\USERNAME) on which the rights are added. True
rights FileSystemRights Rights to add. False
inheritance InheritanceFlags Inheritance flags. The defaults are ContainerInherit | ObjectInherit False
propagation PropagationFlags Propagation flags. The default is None False
accessControlType AccessControlType AccessControl type. The default is Allow False

Nested Elements:

<fileset>

Files to apply changes to.

</fileset>

Remarks

Access rules can be added even if they already exists. The Windows API will take care of duplication.

Examples

  • Add an access rule to allow user "DOMAIN\User1" to write files into directory "E:\MyApp\Logs" and all subdirectories. 

    <add-access-rule dir="E:\MyApp\Logs" ntaccount="DOMAIN\User1" rights="Write" propagation="InheritOnly" />

  • Add an access rule to allow user "DOMAIN\User2" to read all the existing files into directory "E:\MyApp\Logs". 

    <add-access-rule ntaccount="DOMAIN\User2" rights="Read">
    <fileset basedir="E:\MyApp\Logs">
        <include name="*" />
    </fileset>
</add-access-rule>

  • Add an access rule to allow user "DOMAIN\User1" Full Control to network share "SERVER\Share" 

    <add-access-rule share="SERVER\Share" ntaccount="DOMAIN\User1" />
